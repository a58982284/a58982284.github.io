树莓派swap分区的正确方法

因为我的树莓派内存只有512M，所以需要对swap分区进行扩容，作为缓存交换区使用。

错误的做法

下面是错误的做法！错误的做法！错误的做法！重要的事情说三遍！！！

一直以来，我以为树莓派的 swap 和普通 linux 的差不多，也就照着某些网站上的方法修改树莓派的交换分区大小。

熟悉的套路开始了！

cd /var

# 先关闭
sudo swapoff /var/swap

# 重新设置大小1M * 1024 = 1G
sudo dd if=/dev/zero of=swap bs=1M count=1024

# 格式化
sudo mkswap /var/swap

#开启
sudo swapon /var/swap

# 查看当前已生效的swap
swapon -s

# 查看当前swap使用情况
free -m

然而重启后，你就会发现树莓派的 swap 又回到还原了。

正确的做法

如果你按照上面的方法做了，只需删除 swap 分区：

$ sudo swapoff /var/swap
$ sudo rm /var/swap

然后开始真正的表演：

sudo vi /etc/dphys-swapfile

CONF_SWAPSIZE=1024

这一步是修改swap分区的大小。然后重新启动dphys-swapfile 文件服务：

sudo /etc/init.d/dphys-swapfile restart

最后查看大小：

$ free -h

              total        used        free      shared  buff/cache   available
Mem:           433M        132M         42M         34M        257M        215M
Swap:          1.0G          0B        1.0G

如上图所示，已经成功更改了swap分区的大小。
